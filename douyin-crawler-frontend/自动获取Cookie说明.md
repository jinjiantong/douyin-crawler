# 🎉 自动获取Cookie功能说明

## ✨ 新功能上线！

现在您可以一键自动获取抖音Cookie，无需手动复制粘贴！

---

## 🚀 使用方法

### 方式一：使用前端界面（推荐）

#### 1. 打开网站
访问 http://localhost:3000

#### 2. 进入账号管理
点击顶部导航的"账号管理"

#### 3. 点击"添加账号"
点击"添加账号"按钮

#### 4. 点击"打开浏览器登录"
在弹出的窗口中，找到醒目的紫色区域"自动获取 Cookie（推荐）"

点击 **"打开浏览器登录"** 按钮

#### 5. 完成登录
系统会自动：
- ✅ 打开Chrome浏览器
- ✅ 导航到抖音登录页面
- ✅ 等待您扫码或账号密码登录

#### 6. 自动获取
登录成功后：
- ✅ 自动提取Cookie
- ✅ 自动关闭浏览器
- ✅ 自动填充到表单
- ✅ 提示"Cookie获取成功"

#### 7. 保存账号
- 输入账号名称（如：我的抖音）
- 点击"添加账号"完成

---

### 方式二：直接运行脚本

如果您想直接运行Cookie获取脚本：

```bash
# 进入后端目录
cd douyin-crawler-frontend/backend

# 运行Cookie获取脚本
python douyin_cookie_fetcher.py
```

脚本会自动：
1. 打开Chrome浏览器
2. 显示登录页面
3. 等待您完成登录
4. 自动提取Cookie
5. 保存到文件

## 🔧 技术原理

### 工作流程

```
1. 用户点击按钮
   ↓
2. 前端调用API
   ↓
3. 后端启动Selenium
   ↓
4. Selenium打开Chrome
   ↓
5. 导航到抖音登录页
   ↓
6. 用户完成登录
   ↓
7. 检测URL变化
   ↓
8. 提取Cookie
   ↓
9. 保存到文件
   ↓
10. 前端轮询检测
    ↓
11. 自动填充表单
```

### 使用技术

- **Selenium** - 浏览器自动化框架
- **undetected-chromedriver** - 反检测Chrome驱动
- **Flask API** - 后端接口服务
- **轮询机制** - 前端实时检测

## 🆘 常见问题

### Q1: 点击按钮没有反应？

**检查：**
- 确保后端服务正在运行
- 确保端口未被占用
- 检查浏览器是否拦截弹窗

**解决方法：**
```bash
# 重启后端服务
cd Crawler-main
.\venv\Scripts\python.exe main.py
```

### Q2: 浏览器打开但无法加载页面？

**可能原因：**
- 网络问题
- 抖音服务器维护
- 防火墙拦截

**解决方法：**
- 检查网络连接
- 稍后重试
- 使用VPN

### Q3: 登录成功后没有自动获取Cookie？

**可能原因：**
- 登录超时
- 检测机制失效

**解决方法：**
- 等待30秒看是否自动处理
- 点击"取消"重新操作
- 使用手动方式获取Cookie

### Q4: 如何确认Cookie获取成功？

**成功标志：**
- ✅ 浏览器自动关闭
- ✅ 弹窗显示"Cookie获取成功"
- ✅ Cookie输入框已填充内容
- ✅ 出现绿色成功提示

### Q5: Cookie有效期是多久？

**说明：**
- Cookie有效期通常为几天到几周
- 失效后需要重新获取
- 建议定期更新Cookie

## 📝 手动获取备用方案

如果自动获取失败，可以使用手动方式：

### 步骤1：获取Cookie
1. 打开抖音网页版并登录
2. 按 F12 打开开发者工具
3. 点击 Network 标签
4. 进入任意视频详情页
5. 找到 `aweme/detail` 请求
6. 复制 Request Headers 中的 Cookie

### 步骤2：粘贴到表单
1. 在弹窗中粘贴Cookie
2. 输入账号名称
3. 点击"添加账号"
