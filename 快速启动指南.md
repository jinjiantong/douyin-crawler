# 🚀 抖音信息采集网站 - 快速启动指南

## 📋 前置准备

### 1. 确保后端服务正在运行

在启动前端网站之前，需要确保 Crawler 后端服务正在运行：

```bash
# 进入 Crawler-main 目录
cd Crawler-main

# 启动后端服务
.\venv\Scripts\python.exe main.py
```

后端服务运行地址：`http://localhost:8080`

---

## 🎯 快速启动（推荐方式）

### Windows 用户

双击运行根目录的 `启动前端.bat` 文件

### 或手动启动

```bash
# 1. 进入前端目录
cd douyin-crawler-frontend

# 2. 安装依赖（如未安装）
npm install

# 3. 启动开发服务器
npm run dev
```

启动成功后，访问 **http://localhost:3000**

## 📖 使用教程

### 第一步：添加 Cookie

1. 打开网站首页
2. 点击"账号管理"
3. 点击"添加账号"
4. 输入账号名称（如：我的抖音）
5. 粘贴抖音 Cookie
6. 点击"添加账号"

**如何获取 Cookie？**
- 打开抖音网页版并登录
- 按 F12 打开开发者工具
- 点击 Network 标签
- 进入任意视频详情页
- 找到 `aweme/detail` 请求
- 复制 Request Headers 中的 Cookie

### 第二步：搜索视频

1. 点击"搜索视频"
2. 输入关键词（如：美食）
3. 点击搜索按钮
4. 浏览搜索结果
5. 点击视频卡片查看详情

### 第三步：采集评论

1. 打开视频详情页
2. 点击"查看评论"
3. 点击"开始采集评论"
4. 等待数据加载
5. 点击"加载更多"获取更多评论
6. 选择"导出JSON"或"导出CSV"

### 第四步：查看用户信息

1. 在视频详情页点击作者头像/名称
2. 查看用户详细信息
3. 浏览用户作品列表
4. 点击作品查看详情

## 🔧 常见问题

### Q1: 前端启动失败？

**解决方法：**
```bash
# 1. 检查 Node.js 是否安装
node --version

# 2. 如果未安装，下载安装 Node.js
# https://nodejs.org/

# 3. 重新安装依赖
cd douyin-crawler-frontend
npm install
```

### Q2: 后端服务无法连接？

**解决方法：**
```bash
# 1. 确保后端服务正在运行
cd Crawler-main
.\venv\Scripts\python.exe main.py

# 2. 检查后端服务端口
# 默认：http://localhost:8080

# 3. 检查防火墙设置
```

### Q3: 搜索没有结果？

**可能原因：**
- Cookie 已过期
- 网络请求被限制
- 关键词无相关视频

**解决方法：**
1. 更新 Cookie
2. 更换关键词
3. 等待一段时间后重试

### Q4: 评论采集失败？

**可能原因：**
- Cookie 权限不足
- 视频评论已关闭
- 请求频率过高

**解决方法：**
1. 使用有效 Cookie
2. 降低请求频率
3. 稍后重试

## 🌟 高级功能

### 批量采集

如需批量采集多个视频的评论，可以使用 Python 脚本：

```python
import requests
import time

BASE_URL = "http://localhost:8080"
video_ids = ["视频ID1", "视频ID2", "视频ID3"]

for video_id in video_ids:
    # 获取评论
    response = requests.get(f"{BASE_URL}/douyin/comments", params={
        "id": video_id,
        "limit": 100
    })
    
    if response.json()['code'] == 0:
        comments = response.json()['data']['comments']
        print(f"视频 {video_id} 获取到 {len(comments)} 条评论")
    
    time.sleep(2)  # 避免请求过快
```

## ⚠️ 注意事项

1. **Cookie 有效期**
   - Cookie 通常有效期为几天到几周
   - 失效后需要重新获取

2. **请求频率**
   - 建议每次请求间隔 2-3 秒
   - 避免短时间内大量请求

3. **数据使用**
   - 仅供学习和研究使用
   - 请遵守相关法律法规
   - 尊重用户隐私

4. **系统要求**
   - Node.js 16+
   - Python 3.8+
   - 现代浏览器（Chrome、Firefox、Edge）
