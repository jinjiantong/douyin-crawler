# 🔧 后端服务启动说明

## ⚠️ 问题说明

由于某些原因，后端服务启动遇到了问题。请按照以下步骤手动启动：

---

## 📋 启动步骤

### 方法一：使用系统 Python（推荐）

#### 1. 打开终端
在 Crawler-main 目录下打开终端

#### 2. 使用系统 Python 运行
```bash
python main.py
```

或者如果系统 Python 有 fastapi：
```bash
C:\Program Files\Python311\python.exe main.py
```

---

### 方法二：重新创建虚拟环境

如果虚拟环境损坏，可以重新创建：

```bash
# 删除旧虚拟环境
rd /s /q venv

# 创建新虚拟环境
python -m venv venv

# 激活虚拟环境
venv\Scripts\activate

# 安装依赖
pip install fastapi uvicorn pyyaml selenium undetected-chromedriver

# 运行
python main.py
```

---

### 方法三：直接运行已有的后端

如果您之前的后端服务还在运行，可以：

#### 1. 检查是否有旧进程
```bash
netstat -ano | findstr 8080
```

#### 2. 如果有，查看进程 ID
```
TCP    0.0.0.0:8080    0.0.0.0:0    LISTENING    [PID]
```

#### 3. 如果是旧的 Python 进程，停止它
```bash
taskkill /F /PID [PID]
```

#### 4. 重新启动
```bash
python main.py
```

---

## 🧪 测试 API

启动后，可以测试以下 API：

### 1. 检查服务是否运行
```bash
curl http://localhost:8080/docs
```

### 2. 测试抖音账号列表
```bash
curl http://localhost:8080/douyin/account_list
```

### 3. 测试 Cookie API 状态
```bash
curl http://localhost:8080/cookie/status
```

### 4. 测试 Cookie 获取
```bash
curl -X POST http://localhost:8080/cookie/fetch
```

## 🔍 常见问题

### Q1: 提示 "No module named 'fastapi'"
**解决方法：**
```bash
pip install fastapi uvicorn
```

### Q2: 提示 "No module named pip"
**解决方法：**
```bash
python -m ensurepip --upgrade
```
或使用系统 Python。

### Q3: 端口被占用
**解决方法：**
```bash
# 查看占用端口的进程
netstat -ano | findstr 8080

# 停止该进程
taskkill /F /PID [进程ID]
```

### Q4: 虚拟环境无法激活
**解决方法：**
```bash
# 重新创建虚拟环境
python -m venv venv
```

3. **前端服务端口**：3000

## ✅ 验证步骤

后端服务正常启动后，您应该看到：

```
INFO:     Uvicorn running on http://0.0.0.0:8080 (Press CTRL+C to quit)
INFO:     Application startup complete.
```

然后您可以在浏览器访问：
- http://localhost:8080/docs （API 文档）
- http://localhost:3000 （前端网站）
