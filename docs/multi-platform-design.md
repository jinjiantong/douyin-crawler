## 1. 项目概述

### 1.1 项目名称
- **新名称**: 多平台数据采集系统
- **英文名**: Multi-Platform Crawler
- **原名称**: 抖音评论采集系统

### 1.2 项目目标
将现有的抖音评论采集系统扩展为支持多个主流自媒体平台的数据采集系统，同时确保抖音功能的稳定性和完整性不受影响。

### 1.3 支持平台

| 平台 | 状态 | 功能支持 |
|------|------|----------|
| 抖音 | ✅ 已完成 | 搜索、详情、评论、二级评论、主播主页 |
| 小红书 | 🚧 开发中 | 界面展示，逻辑待开发 |

## 2. 设计方案

### 2.1 架构选择

**采用方案 A：平台卡片式首页 + 独立模块**

```
┌─────────────────────────────────────────────────────────────┐
│                   多平台数据采集系统                         │
│                  Multi-Platform Crawler                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│    ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐     │
│    │  抖音   │  │ 小红书  │  │   B站   │  │  快手   │     │
│    │  ✅活跃  │  │  🚧开发  │  │  🚧开发  │  │  🚧开发  │     │
│    └─────────┘  └─────────┘  └─────────┘  └─────────┘     │
│                                                             │
│    ┌─────────┐  ┌─────────┐  ┌─────────┐                   │
│    │  微博   │  │  贴吧   │  │  知乎   │                   │
│    │  🚧开发  │  │  🚧开发  │  │  🚧开发  │                   │
│    └─────────┘  └─────────┘  └─────────┘                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 设计原则

1. **隔离性**: 抖音功能模块完全独立，不受其他平台影响
2. **扩展性**: 新增平台只需添加卡片和路由
3. **一致性**: 所有平台使用统一的设计语言和交互模式
4. **渐进性**: 先上线界面，后续逐步实现各平台功能

## 3. 前端设计

### 3.1 页面结构

```
├── 平台选择页 (PlatformSelect)
│   ├── 顶部导航栏（Logo + 系统名称 + 服务状态）
│   ├── 平台卡片网格（7个平台）
│   └── 底部版权信息
│
└── 平台详情页 (PlatformModule)
    ├── 返回按钮
    ├── 平台标题栏
    ├── 功能导航（Cookie管理、搜索、链接采集、多视频采集）
    └── 功能内容区
```

### 3.2 路由设计

```
/                     → 平台选择页
/douyin/*            → 抖音模块
/xhs/*               → 小红书模块（占位）
/bilibili/*          → B站模块（占位）
/kuaishou/*          → 快手模块（占位）
/weibo/*             → 微博模块（占位）
/tieba/*             → 贴吧模块（占位）
/zhihu/*             → 知乎模块（占位）
```

### 3.3 平台卡片设计

每个平台卡片包含：
- 平台图标/Logo
- 平台名称
- 状态标识（✅已上线 / 🚧开发中）
- 点击跳转功能

### 3.4 视觉风格

**科技深色风**
- 主背景: `#0f172a` (深蓝灰)
- 卡片背景: `#1e293b` (灰色)
- 主色调: `#6366f1` (靛蓝色)
- 强调色: `#ec4899` (粉红色)
- 文字色: `#f1f5f9` (浅白)

## 4. 后端架构

### 4.1 现有架构

后端已支持多平台架构：
```
Crawler-main/service/
├── douyin/      # 抖音（完整实现）
├── xhs/         # 小红书（完整实现）
├── bilibili/    # B站（完整实现）
├── kuaishou/    # 快手（完整实现）
├── weibo/       # 微博（完整实现）
├── tieba/       # 贴吧
└── zhihu/       # 知乎
```

### 4.2 API 设计

各平台 API 保持现有的接口设计：
- `/api/{platform}/search` - 搜索
- `/api/{platform}/detail` - 详情
- `/api/{platform}/comments` - 评论
- `/api/{platform}/replys` - 回复
- `/api/{platform}/user` - 用户信息
